Projeto final - UI Style Guide

Lia Simplis

Guia de estilo de interface para um aplicativo educativo e acessivel sobre seguranca digital.

Este documento define as regras visuais que devem orientar telas, fluxos, componentes e futuras evolucoes do projeto Lia Simplis.

1. Identidade visual

O Lia Simplis usa uma linguagem visual calma, didatica e confiavel. O verde comunica seguranca e acolhimento; o creme reduz a sensacao de interface tecnica; e as cores de apoio ajudam a destacar avisos, ajuda e progresso.

2. Cores principais e regra 60/30/10

Exemplos de uso

Fundo principal: creme #F5F3EF. Cards internos: branco #FFFFFF. Texto principal: #1A1A1A. Texto auxiliar: #666660.

Regra pratica: evitar telas dominadas por amarelo, azul ou vermelho. Essas cores devem aparecer apenas para orientar a leitura ou indicar estado.

3. Iconografia

Estilo recomendado: icones Material minimalistas em linha, com traco simples, cantos suaves e preenchimento apenas quando houver estado selecionado. Os icones devem ter tamanho padrao de 22 a 28 px e cor verde escuro quando ativos.

4. Componentes principais

Botoes

Normal Hover Secundario

Altura minima: 52 px. Raio: 18 px. Texto em negrito para reforcar clareza e acessibilidade.

Campos de formulario

Digite uma palavra ou pergunta
Campo em foco

O estado normal usa borda cinza clara; o foco usa borda verde escuro para indicar interacao ativa.

Cards

Aprenda a identificar golpes

Card branco com borda cinza clara, raio de 18 px e texto direto. Deve agrupar uma acao ou informacao por vez.

Estado destacado

Para dicas, progresso ou informacoes positivas, usar fundo verde palido e borda verde clara.

Barra de navegacao

Inicio    Treino    Ajuda    Dicionario    Salvos

A navegacao inferior deve manter fundo branco, borda superior cinza e item ativo com fundo verde claro.

5. Tipografia

Fonte principal: Inter. Caso a fonte nao esteja disponivel, usar Arial como alternativa. A tipografia deve priorizar leitura rapida, boa legibilidade e textos objetivos.
